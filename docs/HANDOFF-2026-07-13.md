## 6. ★ PRIMARY MISSION: make Flux *regional* inpainting good ★

### Where it lives
- Entry: `src/inpaint.py` — `build_union_mask()` builds ONE mask of everything removed;
  `_inpaint_single_pass()` runs it in ONE Flux/LaMa/OpenCV pass over the whole canvas; `_flux_comfy_inpaint`
  → `qwen_worker.flux_inpaint(rgb, mask, cfg)` posts to ComfyUI. Regional logic should wrap/replace the
  single-pass call in `reconstruct.py`'s inpaint step (find where `background_clean.png` is produced).
- The Flux call itself is proven and fast; the problem is **what you feed it**.

### What I tried and what happened (learn from this)
1. **Whole-image union mask** → Flux ghosts (flat UI ad9) or leaves a soft product ghost (ad2). It sees
   the whole busy canvas and regenerates plausible content.
2. **Regional attempt** (`scratchpad/test_flux_regional.py`): dilate mask 18px → connected components →
   crop padded bbox → inpaint each → composite. FAILED because 18px bridging merged product+headline into
   one **1040×1424** region (≈ whole thing). On that big hole Flux **hallucinated a chocolate-milk glass,
   a herb sprig, and a metal object.** Small regions (text, 256–736px) inpainted fine.

### The core truth (design around this)
Flux Fill **hallucinates plausible OBJECTS when given a large hole on a plain background**, and cleanly
continues background only for **small** holes or **photographic/complex** surroundings. So:
| element | right tool |
|---|---|
| text lines / small elements | Flux small-crop OR simple fill OR LaMa (all fine, fast) |
| **big object on plain/gradient bg** (product pack) | **LaMa** (smooth continuation) — NOT Flux |
| object on photographic/complex bg | **Flux Fill** (its actual strength) |

### Concrete plan to make regional good
1. **Per-element regions, not over-bridged.** Use the fused element boxes (`fused_elements.json` /
   reconstruct entities) directly, or connected components with SMALL bridging (~4–8px, not 18). One crop
   per element, padded ~64–96px so Flux sees real local background.
2. **Route per region by size + background complexity:**
   - small region (area < ~2% canvas, or thin text) → Flux small-crop (guidance ~3.5) or fast fill.
   - big region on a LOW-texture local background (measure local std/gradient around the hole) → **LaMa**,
     not Flux. (LaMa = `simple_lama` in inpaint.py; see §7 — it's currently NOT being selected!)
   - big region on HIGH-texture/photographic local background → Flux.
3. **Feed background context, not the whole canvas.** Crop tightly; the padding IS the context. Consider a
   per-region prompt seeded from the dominant local background color ("clean <color> background").
4. **Composite carefully:** paste back only masked pixels, feather edges (there's `feather_mask_edges` in
   inpaint.py), process largest/most-confident regions first so later crops see cleaned context.
5. **Consider partial denoise** (<1.0) for regions where you want to preserve some structure.
6. Validate on: `runs/inspo_test2` (product, choco bar), `runs/inspo_test`/`inspo_test3`. Compare against
   whole-image Flux and against a PROPERLY-run LaMa.
