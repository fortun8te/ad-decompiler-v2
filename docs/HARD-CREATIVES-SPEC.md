## Capability gaps these demand (beyond the current 002-class fixes)

1. **Peeling / complex layering (TOP PRIORITY — user-mandated "really make it better"):** overlays sitting ON photo content (H1 pills, H4 cards) must peel off leaving a clean photo plate; occlusion-attributed decomposition must handle multiple overlapping occluders (H4: card over card over photo). Text with stuff over it / partially occluded text must be reconstructed as full native text where legible.
2. **Rounded-plate overlay family:** detect + emit rounded rects (corner radius estimation), stadium pills, banner strips, IG-style cards — as SOLID native rects with radius, never raster slices (flat chrome = SOLID per contract).
3. **Emoji-in-text:** leading emoji in pill text (H1) = image chips beside native TEXT (contract), positioned inline correctly.
4. **Instagram-like text:** per-line background-plate text (IG story text style — each wrapped line carries its own rounded background bar hugging the line width; also the AMA widget family). Reference: Michael's Figma node 15510-130 in the Simpletics Creative file (fetch requires Figma auth; implement the standard construct: group = per-line rounded rects + single TEXT node or per-line TEXT, radius ~¼ line height, plate hugs ink + padding).
5. **Ellipse-masked insets:** circular photo insets (H4) = ellipse mask + image fill (we already do this for avatars — must trigger reliably).
6. **Clean cutouts / background removal:** product tubes and boxes photographed on busy backgrounds need high-quality alpha cutouts when they're emitted as separate elements (H2, H5): tight edges, no white fringe/halo (the 002 audit showed doubled contours), color decontamination, soft-shadow handling (shadow either baked into plate or separate soft-shadow element — never half-cut).
7. **Hands/annotation judgment:** hand-held product (H3, H4 inset) — the pipeline must DECIDE: in-scene photo content stays in the photo plate; only cleanly-separable studio products become cutouts. No mangled half-cut hands.
8. **Serif vs sans font classing (H2, H3, H4):** editorial serif headlines must map to a serif Google face, not a sans.
9. **Text directly on photos (H7, H13):** headline/labels sitting on busy or dark photo content with no plate — peel the text off, inpaint the photo under it, emit native TEXT above the plate. This is the make-or-break case for peel.
10. **Chat/social UI family (H9, H12, H14, H16):** DM bubbles (asymmetric radii, gradient fills), avatars (ellipse masks), reply-quote constructs, "New Messages" dividers, tweet rows with engagement glyph chips + native count text, thread lines, screenshot cards separating cleanly from white OR black page backgrounds, redaction blobs kept as chips.
11. **Annotation/decoration family (H10, H11, H13, H17):** scribble strikethroughs over native text, hand-drawn curved arrows, brush-stroke banners (text stays native on top), starburst seals with rotated text, circle badges, callout leader-lines with dot endpoints, circled icon glyphs. Policy: decoration = vector stroke (vectorize path exists) or clean chip; the TEXT it decorates is always native.
12. **Gradient blobs (H8, H17):** soft radial/linear gradient washes emitted as native gradient fills — never traced, never sliced.
13. **Charts (H10):** slice-v1 policy, but ONE clean chart slice with overlapping products excluded via ownership, axis/series labels native where cleanly separable.
14. **Mixed inline styles (H7):** bold/italic segments inside a line = sibling TEXT nodes (contract), preserving baseline.
15. **Non-English text (H17):** accents/diacritics survive OCR→emit (French, etc.).

## Acceptance (per ad class)

Contract standard applies: every string native TEXT (correct class font), flat chrome SOLID (with radius), photo plates intact under peeled overlays (no ghost text/pills baked in), emoji chips, ellipse masks where circular, honest QA (no host-raster escape hatch), "fixable in 15 min" in Figma.
